# 配额与限制

## 已执行限制

当前代码执行的是资源和运行时安全边界，而不是用户配额系统：客户端认证、用户/代理归属、代理 enabled 状态、provider 会话类型、provider open 超时、入口 listener 冲突、HTTPS 证书健康检查和控制通道 TLS 校验均会拒绝不合法操作。

基础 TCP/UDP/HTTP 统计只表示计数能力，不表示配额或限速执行。

## 已知缺口

以下能力当前没有实现，不应从统计或管理字段推断出来：

- 用户或代理级代理数量、并发连接、端口范围和流量配额。
- 带宽上限、实时限速、超额拒绝、暂停和周期结算。
- 配额拒绝的完整错误分类、指标和告警。
- 1C1G、800+ 并发和文件描述符/内存容量目标的验证。

SDK 桥接目前只有 provider open 超时和明确的调用边界；全局、用户、连接级资源限制仍需后续实现并补充测试。相关代码入口包括 `internal/control/`、`internal/session/` 和 `internal/proxy/tunnel/`。
