# 用户列表页 UI 设计

## 1. 页面定位

用户列表页用于集中查看和管理系统中的用户资源。

## 2. 路由

- ` /users `

## 3. 页面目标

- 快速浏览用户信息
- 支持搜索、筛选、排序与分页
- 提供创建用户入口
- 支持进入用户详情页

## 4. 页面结构

- 页面标题区
- 工具栏
- 筛选栏
- 用户表格区
- 分页区

## 5. 工具栏设计

工具栏包含：

- 搜索框
- 角色筛选
- 状态筛选
- 排序控件
- 刷新按钮
- “创建用户”按钮

## 6. 列表字段

- 用户 ID
- 用户名
- 角色
- 状态
- 客户端数
- 代理数
- 上传流量
- 下载流量
- 最近活动时间
- 创建时间
- 更新时间

## 7. 布局设计

- 桌面端以数据表格为主
- 移动端降级为卡片列表
- 工具栏在窄屏下允许换行
- “创建用户”按钮固定为主要操作

## 8. 交互设计

- 输入关键词后支持搜索
- 筛选条件变化时刷新列表
- 点击列头可切换排序
- 点击行进入 ` /users/:id `
- 点击“创建用户”打开弹窗或右侧抽屉
- 创建成功后刷新当前列表

## 9. 弹窗设计

创建用户弹窗包含：

- 用户名
- 初始密码
- 提交按钮
- 取消按钮
- 字段级错误提示

## 10. 状态设计

### 10.1 初始加载
- 表格骨架屏

### 10.2 空数据态
- 系统尚未创建任何用户
- 提示管理员创建第一个用户

### 10.3 无结果态
- 当前筛选条件下无匹配用户
- 提供“清空筛选”动作

### 10.4 创建失败态
- 表单字段错误显示在对应字段下
- 通用错误显示在弹窗顶部

### 10.5 列表错误态
- 显示“用户列表加载失败”
- 保留刷新入口

## 11. 可用性要求

- 列宽合理，避免时间列挤压主要字段
- 用户状态使用标签形式区分
- 搜索与筛选条件在返回列表后保持不丢失

## 12. 验收标准

- 支持分页、搜索、筛选、排序
- 支持创建用户
- 支持进入详情页
- 空态、无结果态、错误态区分明确
