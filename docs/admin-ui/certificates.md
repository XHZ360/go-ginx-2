# 证书页 UI 设计

## 1. 页面定位

证书页用于展示 HTTPS 托管证书状态，并执行签发和续签操作。

## 2. 路由

- ` /certificates `

## 3. 页面目标

- 浏览托管证书生命周期状态
- 支持搜索、筛选、排序和分页
- 支持签发与续签操作

## 4. 页面结构

- 页面标题区
- 工具栏
- 证书列表区
- 分页区

## 5. 工具栏设计

包含：

- 搜索框
- 状态筛选
- 排序控件
- 手动刷新按钮

## 6. 列表字段

- ProxyID
- CertificateID
- Host
- 状态
- 到期时间
- 最近签发时间
- 最近续签时间
- 最近错误
- 创建时间
- 更新时间

## 7. 布局设计

- 桌面端以状态型表格为主
- 状态列与错误列视觉优先级较高
- 移动端可转为证书状态卡片

## 8. 交互设计

- 页面进入后请求证书状态列表
- 默认手动刷新或低频轮询
- 操作期间可短时提升刷新频率
- 每条记录支持“签发”和“续签”操作
- 成功后刷新当前页数据

## 9. 状态设计

### 9.1 初始加载
- 表格骨架屏

### 9.2 空数据态
- 暂无托管证书

### 9.3 无结果态
- 当前筛选条件无匹配证书

### 9.4 操作失败
- 显示签发失败或续签失败原因

### 9.5 列表错误态
- 显示“证书状态加载失败”

## 10. 安全要求

- UI 不展示任何私钥材料
- 错误提示不得泄露敏感文件内容

## 11. 可用性要求

- 到期时间需高可读显示
- 即将过期状态可增加弱提醒样式
- 操作按钮需根据当前状态控制可用性

## 12. 验收标准

- 能展示证书状态列表
- 能发起签发和续签
- 不依赖单独详情页
- 不暴露私钥或敏感凭据
