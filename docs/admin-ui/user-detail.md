# 用户详情页 UI 设计

## 1. 页面定位

用户详情页用于查看单个用户信息，并执行禁用和改密等管理动作。

## 2. 路由

- ` /users/:id `

## 3. 页面目标

- 展示用户基础信息与状态
- 展示用户资源概览
- 支持禁用用户
- 支持修改用户密码

## 4. 页面结构

- 返回导航区
- 用户基础信息卡片
- 用户状态卡片
- 资源统计卡片
- 操作区

## 5. 信息展示

- 用户 ID
- 用户名
- 角色
- 状态
- 客户端数量
- 代理数量
- 最近活动时间
- 创建时间
- 更新时间

## 6. 布局设计

- 顶部显示返回列表入口与页面标题
- 中部使用 2 到 3 个信息卡片分区
- 操作按钮区域与信息区分离，避免误触
- 移动端按卡片垂直堆叠

## 7. 交互设计

- 页面进入后请求用户详情
- 点击“禁用用户”前需二次确认
- 点击“修改密码”打开弹窗
- 操作成功后刷新详情内容

## 8. 修改密码弹窗

包含：

- 新密码输入框
- 确认密码输入框
- 提交按钮
- 取消按钮

规则：

- 两次密码输入必须一致
- 不回显任何已有密码信息

## 9. 状态设计

### 9.1 初始加载
- 卡片骨架屏

### 9.2 资源不存在
- 显示“用户不存在或已删除”

### 9.3 操作失败
- 禁用失败或改密失败需展示明确提示

### 9.4 会话过期
- 返回登录页

## 10. 可用性要求

- 重要动作按钮需与普通浏览信息有明显区分
- 危险操作使用风险色
- 详情页返回列表后应保留原列表筛选状态

## 11. 验收标准

- 能正确显示用户详情
- 能执行禁用与修改密码
- 不展示明文密码或敏感密码衍生信息
