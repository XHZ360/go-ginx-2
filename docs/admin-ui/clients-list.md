# 客户端列表页 UI 设计

## 1. 页面定位

客户端列表页用于展示受管客户端及其实时运行状态。

## 2. 路由

- ` /clients `

## 3. 页面目标

- 浏览客户端静态信息和在线状态
- 提供检索、筛选、排序与分页能力
- 支持进入客户端详情页

## 4. 页面结构

- 页面标题区
- 工具栏
- 列表区
- 分页区

## 5. 工具栏设计

包含：

- 搜索框
- 用户筛选
- 状态筛选
- 在线状态筛选
- 排序控件
- 手动刷新按钮

## 6. 列表字段

- 客户端 ID
- 所属用户
- 名称
- 状态
- 版本
- 最近在线时间
- 最近离线时间
- 在线状态
- 当前协议
- 活跃代理数
- 活跃流数
- 上传流量
- 下载流量
- 错误摘要

## 7. 布局设计

- 桌面端使用表格布局
- 在线状态与错误摘要需有足够视觉区分
- 移动端转为卡片流式布局
- 工具栏在窄屏下换行排列

## 8. 交互设计

- 页面进入后拉取列表数据
- 维持 5 秒轮询，刷新运行态字段
- 点击表格行进入 ` /clients/:id `
- 更改筛选和分页时仅刷新当前查询

## 9. 状态设计

### 9.1 初始加载
- 表格骨架屏

### 9.2 空数据态
- 系统暂无客户端

### 9.3 无结果态
- 当前条件下无匹配客户端

### 9.4 轮询失败
- 保留旧数据，并提示最近刷新失败

### 9.5 列表错误态
- 首次加载失败时显示错误面板

## 10. 可用性要求

- 在线状态使用颜色和文字双重表达
- 协议、状态、版本等字段在窄列下保持可读
- 列表刷新不应造成明显闪烁

## 11. 验收标准

- 支持分页、搜索、筛选、排序
- 支持 5 秒轮询
- 支持跳转详情
- 错误态与轮询失败态区分明确
