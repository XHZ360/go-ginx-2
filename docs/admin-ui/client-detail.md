# 客户端详情页 UI 设计

## 1. 页面定位

客户端详情页用于展示客户端身份信息、运行状态以及其管理的代理列表。

## 2. 路由

- ` /clients/:id `

## 3. 页面目标

- 展示客户端基础资料
- 展示当前会话和心跳状态
- 展示关联代理信息
- 支持为当前客户端创建代理
- 支持轮换凭据和完全删除客户端
- 支持从客户端视角查看节点运行情况

## 4. 页面结构

- 返回导航区
- 基础信息卡片
- 运行状态卡片
- 会话信息卡片
- 关联代理列表区

## 5. 信息展示

- 客户端 ID
- 所属用户
- 名称
- 状态
- 版本
- 最近在线时间
- 最近离线时间
- 当前协议
- ConnectedAt
- LastHeartbeat
- ConfigVersion
- ActiveProxies
- ActiveStreams
- 上传流量
- 下载流量
- ErrorSummary

## 6. 关联代理展示

- 以表格或简化列表展示 managed proxies
- 每一项至少显示代理 ID、名称、类型、状态、入口信息
- 点击条目可跳转代理详情页

## 7. 布局设计

- 顶部信息区优先展示身份和在线状态
- 运行指标与会话信息并列展示
- 底部展示关联代理列表
- 移动端统一改为纵向堆叠

## 8. 交互设计

- 页面进入后请求客户端详情
- 维持 5 秒轮询刷新运行态和心跳数据
- 列表中的关联代理支持跳转
- “Create proxy”动作跳转到 ` /proxies?create=1&userId=<userId>&clientId=<clientId> `，代理创建弹窗默认带入当前客户端上下文
- 删除客户端使用确认动作；仍有关联启用代理时必须在详情页展示冲突错误，不丢弃当前详情内容
- 新 credential 展示区域最多显示 3 行，超出内容在区域内滚动

## 9. 状态设计

### 9.1 初始加载
- 详情骨架屏

### 9.2 不存在态
- 显示“客户端不存在”

### 9.3 错误态
- 显示“客户端详情加载失败”

### 9.4 轮询失败
- 保留当前数据并提示刷新失败

## 10. 可用性要求

- 在线信息需突出展示
- 心跳时间和连接时间需使用易读时间格式
- 轮询更新不能打断用户阅读或滚动位置

## 11. 验收标准

- 能显示客户端详情与运行态叠加信息
- 能显示 managed proxies
- 能支持代理详情跳转
- 能从当前客户端进入代理创建流程
- 能删除无启用代理依赖的客户端，并对启用代理冲突给出当前页错误反馈
- 5 秒轮询不重置页面状态
