# 登录页 UI 设计

## 1. 页面定位

登录页用于管理员身份认证，是唯一未登录可访问页面。

## 2. 路由

- ` /login `

## 3. 页面目标

- 让管理员完成账号密码登录
- 检查已有会话并自动跳转后台首页
- 为受保护页面提供统一登录入口

## 4. 页面结构

- 页面容器
- 品牌区
- 登录卡片
- 用户名输入框
- 密码输入框
- 登录按钮
- 错误提示区

## 5. 布局设计

- 桌面端采用居中单卡片布局
- 页面背景保持简洁，弱化装饰，突出登录动作
- 登录卡片宽度控制在 360px 到 420px
- 移动端卡片宽度自适应，左右保留安全边距

## 6. 视觉层级

- 顶部为产品名称与一句简短说明
- 中部为登录表单主体
- 底部可放置环境标识或版权信息，非必需
- 主按钮为页面唯一高优先级操作

## 7. 交互设计

- 页面加载先请求 `GET /api/admin/session`
- 若会话有效，直接跳转 ` /dashboard `
- 提交表单时请求 `POST /api/admin/login`
- 登录中按钮置为 loading 并禁用重复提交
- 登录失败后保留用户名输入内容
- 按回车键可直接提交表单

## 8. 表单规则

- 用户名不能为空
- 密码不能为空
- 字段错误显示在输入框下方
- 认证失败显示在表单顶部错误区

## 9. 状态设计

### 9.1 初始加载
- 表单可见但按钮禁用，或显示轻量加载态

### 9.2 校验失败
- 对应字段展示红色错误提示

### 9.3 认证失败
- 显示统一错误文案，不暴露实现细节

### 9.4 网络失败
- 显示“网络异常，请稍后重试”

### 9.5 已登录
- 不停留在登录页，直接进入后台

## 10. 可用性要求

- 用户名、密码输入框支持浏览器自动填充
- Tab 键顺序正确
- 错误提示与输入框关联
- 移动端点击输入框时布局不抖动

## 11. 验收标准

- 未登录用户可正常输入并提交登录
- 已登录用户访问登录页会自动跳转
- 登录失败可明确感知失败原因
- 页面在桌面端和移动端均可稳定使用
