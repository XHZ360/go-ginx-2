# 审计页 UI 设计

## 1. 页面定位

审计页用于查看最近的管理操作时间线，是轻量级控制面事件视图。

## 2. 路由

- ` /audit `

## 3. 页面目标

- 浏览最近审计事件
- 支持按关键条件筛选事件
- 支持管理员快速定位资源操作记录

## 4. 页面结构

- 页面标题区
- 工具栏
- 审计列表区
- 分页区

## 5. 工具栏设计

包含：

- 搜索框
- ActorType 筛选
- ActorID 筛选
- ResourceType 筛选
- Action 筛选
- Result 筛选
- 排序控件
- 手动刷新按钮

## 6. 列表字段

- Timestamp
- ActorType
- ActorID
- ResourceType
- ResourceID
- Action
- Result
- 摘要信息

## 7. 布局设计

- 桌面端以时间线表格或标准表格展示
- 默认按时间倒序
- Result 和 Action 使用轻量标签区分
- 移动端转为事件卡片列表

## 8. 交互设计

- 页面进入后拉取最近审计记录
- 支持搜索、筛选、排序、分页
- 默认手动刷新或低频刷新
- 不提供导出、日志关联、全文检索

## 9. 状态设计

### 9.1 初始加载
- 列表骨架屏

### 9.2 空数据态
- 暂无审计记录

### 9.3 无结果态
- 当前筛选条件无匹配记录

### 9.4 列表错误态
- 显示“审计记录加载失败”

### 9.5 会话过期
- 跳转登录页

## 10. 可用性要求

- 时间字段支持本地化展示
- 摘要信息需可读，避免只显示机器字段
- 高密度数据场景下仍保持时间与资源字段清晰

## 11. 验收标准

- 能展示最近审计事件
- 默认倒序展示
- 支持基础筛选与分页
- 不扩展为完整 observability 检索界面
