# 代理列表页 UI 设计

## 1. 页面定位

代理列表页用于集中展示和管理 TCP、UDP、HTTP、HTTPS 代理资源。

## 2. 路由

- ` /proxies `

## 3. 页面目标

- 浏览代理资源及运行状态
- 支持创建、筛选、排序与分页
- 提供启停等快速生命周期操作
- 支持跳转代理详情页

## 4. 页面结构

- 页面标题区
- 工具栏
- 代理表格区
- 分页区

## 5. 工具栏设计

包含：

- 搜索框
- 用户筛选
- 客户端筛选
- 类型筛选
- 状态筛选
- 排序控件
- 手动刷新按钮
- “创建代理”按钮

## 6. 列表字段

- 代理 ID
- 名称
- 类型
- 所属用户
- 所属客户端
- 状态
- 运行状态
- EntryBindHost
- EntryHost / Domain
- EntryPort
- TargetHost
- TargetPort
- 活跃连接数
- 上传流量
- 下载流量
- TCP 错误数
- UDP 错误数
- HTTP 错误数
- 创建时间
- 更新时间

## 7. 布局设计

- 桌面端使用宽表格布局
- 状态、类型与运行状态使用标签样式区分
- 移动端转为卡片模式
- 创建按钮固定为主要操作

## 8. 创建代理交互

创建入口使用弹窗或抽屉，按代理类型动态显示字段：

- 通用字段：名称、所属用户、所属客户端、描述
- TCP/UDP：BindHost 选项、EntryPort、TargetHost、TargetPort
- HTTP/HTTPS：BindHost 选项、EntryPort、HTTP Host 或 HTTPS SNI 域名、TargetHost、TargetPort
- HTTPS：可选 CertFile 和 KeyFile
- 当 URL 或路由状态携带 `userId` 和 `clientId` 时，创建表单默认选中对应用户和客户端，并在候选项未加载完成时保留 ID fallback
- 用户和客户端选择器必须允许管理员修改；修改用户后，如果当前客户端不属于新用户，应清空或要求重新选择客户端
- BindHost 必须来自管理员 API 返回的监听地址选项，包含默认监听、wildcard、loopback 和可发现的本机地址

## 9. 列表交互

- 点击行进入 ` /proxies/:id `
- 列表行可提供启用/禁用快捷操作
- 页面保持 5 秒轮询刷新运行态和统计数据
- 创建、更新或启用代理发生监听冲突或 listener 启动失败时，明确展示 `ENTRY_CONFLICT`
- 弹窗蒙版仅在鼠标按下和释放都发生在蒙版层时关闭；从弹窗内按下再在蒙版释放不得关闭

## 10. 状态设计

### 10.1 初始加载
- 表格骨架屏

### 10.2 空数据态
- 暂无代理

### 10.3 无结果态
- 当前筛选条件无匹配代理

### 10.4 校验失败
- 表单字段级错误提示

### 10.5 冲突态
- 对端口冲突单独展示，说明监听已被占用

### 10.6 列表错误态
- 显示“代理列表加载失败”

## 11. 可用性要求

- 类型切换时表单字段变化应平滑明确
- 表格宽字段支持截断和悬浮提示
- 快捷启停操作需避免误触，可使用确认机制

## 12. 验收标准

- 支持四类代理资源浏览
- 支持创建与快速启停
- 支持搜索、筛选、排序、分页
- 对 `ENTRY_CONFLICT` 有清晰 UI 提示
- 支持从客户端页面携带默认 `userId` 和 `clientId` 打开创建代理弹窗
- 创建弹窗在小视口中可内部滚动且不会溢出视口
