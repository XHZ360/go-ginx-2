# 仪表盘 UI 设计

## 1. 页面定位

仪表盘是管理员登录后的默认首页，用于展示当前系统核心运行概览。

## 2. 路由

- ` /dashboard `

## 3. 页面目标

- 快速展示可信的系统核心指标
- 帮助管理员判断整体运行是否正常
- 提供后台其他模块的导航起点

## 4. 页面结构

- 页面标题区
- 指标卡片区
- 次级说明区
- 刷新状态区

## 5. 指标卡片

页面至少展示以下卡片：

- 在线客户端数
- 已启用代理数
- 活跃 TCP 连接数
- 累计上传流量
- 累计下载流量
- TCP 错误数
- UDP 错误数
- HTTP 错误数

## 6. 布局设计

- 桌面端使用多卡片网格布局，优先 4 列或 2 列组合
- 平板端降为 2 列
- 移动端降为 1 列堆叠
- 卡片之间保持统一间距

## 7. 视觉设计

- 指标值为主视觉层级
- 指标标题为次级层级
- 错误类指标可使用提醒色，但整体避免告警中心式夸张设计
- 保留“最近刷新时间”作为弱信息

## 8. 交互设计

- 页面加载后请求 GraphQL 仪表盘汇总数据
- 页面保持 5 秒轮询刷新
- 提供手动刷新按钮
- 刷新时仅更新卡片数据，不重置页面壳层

## 9. 状态设计

### 9.1 初始加载
- 卡片骨架屏

### 9.2 零值态
- 若暂无运行数据，卡片显示 0 或空值占位

### 9.3 错误态
- 显示“指标加载失败”，保留手动刷新入口

### 9.4 会话过期
- 终止轮询并跳回登录页

## 10. 可用性要求

- 数字需支持大数格式化
- 流量值需自动转换为易读单位
- 卡片标题和数值在窄屏下不换行错位

## 11. 验收标准

- 页面能稳定展示 8 个核心指标
- 5 秒轮询只影响当前页面数据
- 桌面端和移动端布局均清晰可读
